# Evaluación de Conocimientos: GitHub y Markdown
 
## 1. Conceptos Generales
**¿Qué es un archivo Markdown y para qué sirve?**
> ES un archivo de texto que permite escribir contenido con formato, se utiliza para organizar la infromacion de manera clara 
**¿Cuál es el propósito del archivo README.md en un repositorio de GitHub?**

 El archivo README.md sirve para presentar y explicar la información principal de un proyecto, en este archivo se describe de qué trata el repositorio, cuáles son sus objetivos, cómo funciona y cómo puede ser utilizado por otras personas

## 3. Flujo de Trabajo en GitHub Desktop
**¿Qué diferencia hay entre hacer un "Commit" y hacer un "Push"?**
* **Commit:** (Guarda los cambios realizados en los archivos del repositorio en la computadora, creando un registro de las modificaciones)
* **Push:** (Envía los cambios guardados al repositorio en GitHub en internet para que puedan verse en la plataforma)
 
**¿Para qué sirve el botón "Fetch Origin"?**
> Sirve para buscar y verificar si existen cambios nuevos en el repositorio que está en GitHub, de esta manera se puede mantener actualizado el repositorio local con la versión que está en línea
